Google Groups User Settings Guide

 

While each group is created with a specific set of attributes, many of the default settings can be modified by the groupâ€™s manager. These customizable attributes are accessed from the menu appearing on the left side of the group management screen, and are divided into four categories:

Settings

controls group appearance and behavior with regard to members

Email options
Subject prefix This text will be displayed before whatever you type into the subject of a message.
Email footer What appears beneath each message from the group. By default, includes instructions for using the Google Group. Can also include a custom message with the large text box.
Auto replies Enable automatic replies to emails that the group receives. Can set custom messages for different types of recipients.
Post replies Options for who sees users' replies to list emails.
Max message size Choose a size limit for messages sent to the group. By default, this is 25 MB.
Bounced email notifications Choose whether or not users whose email bounces are notified.

Identity
Required forms of identity Regulates under which identity authors of posts are identified; their chosen display name, their google profile name, or either.
Display names and email addresses On/off setting to require that every user has a unique display name.
Moderation
Moderate messages On/off setting to moderate messages sent to the group or not.
New member restrictions Choose whether or not new users have restrictions placed on their ability to post.
Rejected author notifications Choose whether or not to notify users when moderators reject their post. Can write a custom message to send.
Spam messages Options for what to do with messages that Google automatically flags as spam.

Tags
Post rule On/off setting to restrict whether or not users have to enter at least one tag to post.
Number of tags per topic maximum Choose how many tags users can put on each post. From 1 to 14.
Tag suggestions Suggested tags for users who post to the list to use.

Categories
Add a new category group Create a new category group with a name and short description. Category groups serve as a way to logically group like categories.
Add a new category Create a category with a name and short description. Categories serve as a themed place to host topics of the same nature.

Roles

controls settings and permissions for a named class of users

Google Groups default roles
Name Owner, Manger, Member. Cannot change.
Description Default description is a barebones description of the role and cannot be modified.
Members (who currently hold this role) Searchable list of members with that role. Button allows you to add members to this role with a searchable list of the whole group's members.
Permissions (will affect all users who hold this role) List of permissions for the role. Can be modified via the edit button next to "Permissions."
User generated roles
Name Enter a name for your role.
Description Provide a short description of the role's purpose, possibly add an example of who would hold the role.
Domain restriction Restrict who can have this role based on the domain of their email address. For example, @case.edu or @gmail.com.
Badge Select a photo to be used as the icon for this role. Can choose whether or not to display the badge next to member names and a rank for this role.
Members (who currently hold this role) Searchable list of members with that role. Button allows you to add members to this role with a searchable list of the whole group's members.
Permissions (will affect all users who hold this role) Add privileges for the role. By default the role will not have any special privileges at all. Can be added in the edit menu.

Information

controls how the group interacts with mail servers and appears on the web

General information
Group name Edit the group's name.
Group email address Edit the group's email address.
Group description Provide a description of the purpose of the group.
Group's primary language Drop-down to select the primary language of the group. Changes the language of the pre-made messages included in each email.
Topic types Choose the topic types that are allowed and which one will be selected by default.
Posting options Choose where users can post to the group from.
Embedding your groups HTML that you can copy into a web page's source HTML to embed a link to the group into an external web page.
Directory
Group directory Choose whether or not to list this group in the directory of Case domain google groups.
Content control
Archive options Choose whether or not to archive posts to the group.
Orphaned posts desc
Web view customization
Best answers Choose whether or not to prioritize displaying best answers over other replies to a post.
Conversation mode Choose whether or not emails are grouped into threads based on subject.
Subject length Choose a maximum length for the subject line of posts to the group.
Advanced

The options listed under this heading are controlled by the group owner and are not editable by the group's manager.

Permissions

determines the degree of control a user or role has with regard to group settings

Basic permissions
View topics Choose which roles within the group can view topics.
Post Choose which roles can post to the group./td>
Join the group Change who can join the group and what conditions have to be met to join.
Allow members external to this organiztion Choose whether or not email addresses outside of Case's domain can be members of the group.
New member question Write a question that users who request to join the group can answer to determine whether or not they qualify to join.

Posting permissions
Add references Choose which roles or specific users can add references to topics.
Assign topics Choose which roles can assign topics in a forum to another user.
Attach files Choose which roles can attach files to posts.
Enter free form tags Choose which roles can enter free form tags for topics in a forum.
Have custom profile photos Choose which roles can have custom profile photos.
Mark favorite reply on own topic Choose which roles can mark a reply to one of their topics as a favorite reply.
Me too topic Choose which roles can "bump" a question with the Me Too feature.
Post Choose which roles can post messages to the group.
Post annoucements Choose which roles can post announcements to the group.
Post as the group Choose which roles can anonymously post and reply as the group.
Post rich text format Choose which roles can post in rich text format.
Reply to author Choose which roles can reply privately to the author of a topic.
Reply to auto-closed topic Choose which roles can reply to topics that have been automatically closed.
Use canned responses Choose which roles can use canned responses to reply to topics.
Access permissions
Contact the owners of this group Choose which roles can contact the owner(s) of the group.
View members Choose which roles can view the members of the group.
View member email addresses Choose which roles can view members' email addresses.
View topics Choose which roles can view the topics in the group.

Moderate permissions
Add members Choose which roles can add members to the group.
Approve members Choose which roles can approve requests to join the group.
Approve messages Choose which roles can approve messages from those who cannot post directly to the group.
Ban users Choose which roles can ban and unban users from the group.
Change any tag or category Choose which roles can change tags or categories of posts.
Delete any post Choose which roles can delete any post.
Delete topics Choose which roles can delete topics from the group.
Edit others' posts Choose which roles can edit the text of others' posts.
Edit own posts Choose which roles can edit the text of their own posts.
Hide abuse Choose which roles can hide and report abusive messsages.
Invite members Choose which roles can invite other users to be a member.
Lock topics Choose which roles can lock topics to block replies and prevent edits.
Mark duplicate Choose which roles can mark a topic as a duplicate of another topic.
Mark favorite reply on any other user's topics Choose which roles can mark any other user's post as a favorite reply.
Mark no response needed Choose which roles can flag a topic as not needing a response.
Modify members Choose which roles can modify a member, such as what role they have or turning on targeted moderation.
Move topics in Choose which roles can move topics into the forum.
Move topics out Choose which roles can move topics out of the forum.
Sticky topics Choose which roles can make a topic "sticky" by pinning it to the top of the forum.
Take topic Choose which roles can take a topic in the forum.
Unassign topic Choose which roles can remove an assigned user from a topic.
Unmark favorite reply on any topic Choose which roles can un-pin any favorited post.
