Requirements

High level requirements are numbered with whole integers (e.g., counting numbers - 1, 2, 3, etc.) Detail requirements are associated with one high-level requirement and designated by with an ID number in the format, 'high-level ID.x' (e.g., 1.1, 1.2, 1.3, etc.)

Generally, these requirements describe the behaviors that the new system must exhibit, based on the features and uses of the old system. To review a granular matrixed comparison of requirements between the old and new systems, as well as known, planned upgrades to the new system, see the Requirements Checklist.

ID: 1
Desc: the system shall replicate reporting capabilities available to all users with current system
Rationale: any replacement system must not reduce functionality provided by current system
Fit: all detailed requirements are met by new system

ID: 1.1
Desc: for each individual course section, any user shall be able to review anonymized survey respondent data including:

  • text of each question included in survey
  • description of scale or answer key
  • description of conversions to numerical scores and averages calculations
  • percentage-wise distribution of question responses by class (section)
  • count distribution of question responses by class (section)
  • class (section) average response for each survey question
  • course (all sections) average response for each survey question
  • overall average (all classes in the same term evaluated using the same questions)
  • response count (section)
  • enrollment count (section)
Rationale: stakeholder requirement
Fit: all listed reporting capabilities are replicated in new system

ID: 1.2
Desc: course sections shall be identified using the following data:

  • semester
  • academic year
  • course number
  • course title
  • section number
Rationale: stakeholder requirement
Fit: all listed reporting capabilities are replicated in new system

ID: 1.3
Desc: section reporting data shall be available to any user, organized by semester, summarized in any of the following groups:

  • all courses
  • School or Dept (e.g., Arts and Sciences, MSASS, Nursing)
  • Level (e.g., 100, 200, 300, 400, 500, 600)
  • Subject (e.g., ACCT, ECON, STAT, etc.)
Rationale: stakeholder requirement
Fit: all listed reporting capabilities are replicated in new system

ID: 2
Desc: the system shall replicate reporting capabilities available to instructors using the current system
Rationale: any replacement system must not reduce functionality provided by current system
Fit: all detailed requirements are met by new system

ID: 2.1
Desc: for each section taught, an instructor shall be able review each submitted evaluation individually
Rationale: stakeholder requirement
Fit: all listed reporting capabilities are replicated in new system

ID: 2.2
Desc: for each evaluation submitted for a section taught, an instructor shall be able to review the full text of respondent comments
Rationale: stakeholder requirement
Fit: all listed reporting capabilities are replicated in new system

ID: 2.3
Desc: for each individual course section, any instructor shall be able to review anonymized survey respondent data including:

  • text of each question included in survey
  • description of scale or answer key
  • description of conversions to numerical scores and averages calculations
  • percentage-wise distribution of question responses by class (section)
  • count distribution of question responses by class (section)
  • class (section) average response for each survey question
  • course (all sections) average response for each survey question
  • overall average (all classes in the same term evaluated using the same questions)
  • response count (course)
  • enrollment count (course)
Rationale: stakeholder requirement
Fit: all listed reporting capabilities are replicated in new system

ID: 2.4
Desc: course section reporting shall provide the following identifiable data:

  • semester
  • academic year
  • course number
  • course title
  • section number
Rationale: sections must be indentifiable in familiar format and with consistent data points
Fit: all listed reporting capabilities are replicated in new system

ID: 2.5
Desc: for team-taught courses, course evaluation responses shall be produced for each instructor
Rationale: stakeholder requirement
Fit: all listed reporting capabilities are replicated in new system

ID: 3
Desc: the system shall replicate reporting capabilities available to academic administrators using the current system
Rationale: any replacement system must not reduce functionality provided by current system
Fit: all detailed requirements are met by new system

ID: 3.1
Desc: the system shall provide the capability to review instructor performance over time
Rationale: stakeholder requirement, used for tenure reviews and similar
Fit: all listed reporting capabilities are replicated in new system

ID: 3.2
Desc: the system shall allow administrative users to export raw data
Rationale: several stakeholders use data exported from the current system to construct reports outside of the system
Fit: raw data is available for export by administrative users

ID: 3.3
Desc: data exported from the system shall be analagous to data values exported from the current system
Rationale: stakeholders who require raw data for external reporting must be able to compare newly created data to historical data
Fit: raw data exported from system is similar in content to current data exports

ID: 4
Desc: the system shall not present a detailed section report publicly if the enrollment is 3 or fewer
Rationale: policy constraint of current system
Fit: sections with low enrollment are not reported individually

ID: 5
Desc: the system shall aggregate results for cross-listed sections and report them together in any cross-listed section a user selects
Rationale: any replacement system must not reduce functionality provided by current system
Fit: results for cross-listed sections are aggregated and available in reports for any selected section
Cross-listings between ungregraduate and graduate level courses are exepmted from this requirement when different survey questions are selected.

ID: 6
Desc: the system shall provide a 'print-ready' format option for all system-generated reports
Rationale: any replacement system must not reduce functionality provided by current system
Fit: sytem-generated reports are formatted for print
Cross-listings between ungregraduate and graduate level courses are exepmted from this requirement when different survey questions are selected.

ID: 7
Desc: the project team shall provide a glossary of terms related to reporting in the new system
Rationale: current and new system refer to similar or same data and workflow using distinct terminology
Fit: glossary with term definitions and synonymous terms from old system is published

Student Reporting

Measure instructor, course, results against one another, or a benchmark group.

  1. Choose area.
  2. Choose instructor(s), or leave default settings (All Instructors).
  3. Choose course(s), or leave default settings (All Courses).
  4. Choose term(s), or leave default settings (All Terms).
  5. Choose year(s), or leave default settings (All Years).
  6. Choose a question on which to report.
  7. Click View Question Response Scale to review answer key.
  8. Click Search.
  9. Matching records are shown in the "Results" list.
  10. Click any column header to sort the list by that criterion. Click the header again to reverse sort order. Criteria are:
    • Course
    • Instructor
    • Term
    • Respondents (total)
    • Average (based on scale)

Response Rate Tracker

Track course evaluation response rate.

  1. Review response rates by project
  2. Choose a project from the list provided or search by name. The following data are provided:
    • Project Name
    • Enrollments (total count)
    • Responded (total count, opt-outs not excluded)
    • Response Rate (percentage, opt-outs not excluded)
    • Opted-Out (total count)
    • % of Enrollments Opted-Out
    • Responded with Opted-Out Removed (count)
    • Response Rate with Opted-Out Removed (percentage)
  3. Review response rates in a project by course section
  4. Choose a project from the list provided or search by name.
  5. Click the name of the project to open detail.
  6. View courses in the list of results.
  7. If needed, search for a specific course or section using course code, course title or course unique ID. The following data are provided:
    • Code (course identifier)
    • Title (course identifier)
    • Unique ID (section identifier)
    • Enrollments (total count)
    • Responded (total count, opt-outs not excluded)
    • Response Rate (percentage, opt-outs not excluded)
    • Opted-Out (total count)
    • % of Enrollments Opted-Out
    • Responded with Opted-Out Removed (count)
    • Response Rate with Opted-Out Removed (percentage)

Project Results

Preview the results of several, all or individual courses associated with a single project.

  1. Choose a project from the list provided or search by name.
  2. If needed, choose additional limiting criteria (project end date OR results available) and an associated date range.
  3. Click magnifying glass icon in "View" column to open results.
  4. Individual Course Reports
    1. Choose an individual course and click the download icon in "View" column.
    2. Choose applicable format (PDF, PDF+Comments, Raw Data).
    3. System will display message confirming that report is being generated.
    4. Choose to save the report to a location, or open with application.
    5. Open report using appropriate application (e.g., Adobe PDF, Microsoft Excel).
  5. Batch Reporting
    1. To report the results of more than one course together, click the selection box for each and click Batch Report.
    2. Name the report.
    3. Choose a data format option:
      • Merge Multiple PDF Reports into one PDF for Selected Courses
      • Download Multiple PDF Reports/Batch as ZIP file for Selected Courses
      • Aggregate Data for Selected Items into One Report
      • Choose whether or not to include "Write-in Questions"
    4. Click Go.
    5. System will display message confirming that report is being generated.
    6. Report results will be emailed to user from system when complete.

Instructor Results

Results for individual instructors (pre-determined based on current user credentials)

  1. View Setup
    1. All available columns are shown by default. Choose to show or hide any of the following (report results will not be affected):
      • Course Code
      • Title
      • Unique ID
      • Hierarchy Level
      • Project
    2. Choose project search scope. Select from projects from dropdown list named "Project".
    3. Click Search to show courses in selected projects in "Results" list below.
  2. Individual Instructor, Individual Course Reporting
    1. Choose an individual course and click the download icon in "View" column.
    2. Choose applicable format (PDF, PDF+Comments, Raw Data).
    3. System will display message confirming that report is being generated.
    4. Choose to save the report to a location, or open with application.
    5. Open report using appropriate application (e.g., Adobe PDF, Microsoft Excel).
  3. Individual Instructor, Batch Course Reporting
    1. To report the results of more than one course together, click the selection box for each and click Batch Report.
    2. Name the report.
    3. Choose a data format option:
      • Merge Multiple PDF Reports into one PDF for Selected Courses
      • Download Multiple PDF Reports/Batch as ZIP file for Selected Courses
      • Aggregate Data for Selected Items into One Report
      • Choose whether or not to include "Write-in Questions".
    4. Click Go.
    5. System will display message confirming that report is being generated.
    6. Report results will be emailed to user from system when complete.

Report Builder

Build custom reports.
ITS strongly recommends EvaluationKit administrator guidance when using this option.

  1. Click + Create New Report.
  2. Set Report Properties
    Add report name and brief description. Click Continue.
  3. Choose Projects
    1. Select project(s) from "Select Projects" list.
    2. Click + Add Selected Projects.
    3. Verify correct projects appear in "Report Projects" list and click Continue.
  4. Choose Questions to Report
    Choose one or more questions to add to your report and click Continue
  5. Build Your Query
    1. Choose "Areas" (e.g., Engineering, Francis Payne Bolton School of Nursing) to limit query scope.
    2. Click + Add Selected Areas.
    3. Choose limiting criteria for particular questions (e.g., question #1 where responses are selected and the response is greater than x_value).
    4. Click Save.
    5. Click Select Instructors to limit report by specific instructors. Or, leave default value (all instructors).
    6. Click Select Courses to limit report by specific course. Or, leave default value (all courses).
    7. Scroll to top of page and click Continue.
  6. Choose a Report Template
    Click number links at bottom to preview each option:
    • 1. Node Summary Report
    • 2. Node Summary Report with Project Detail
    • 3. Project Summary Report
    • 4. Project Summary Report with Node Detail
    • 5. Course Summary Report
    • 6. Course Summary Report with Project Detail
    • 7. Instructor Summary Report
    • 8. Instructor Summary Report with Project Detail
  7. Click Generate
  8. Results are shown.
    NOTE: If no results match query parameters, "No Data Found" is displayed in red.

Results Feedback

Review survey comments.

  1. Search for comments using one or more of the following criteria:
    • Project
    • Course code
    • Course title
    • Course unique ID
    • Hierarchy level
  2. To review new comments only, check the box named "New Only".
  3. Click Search.
  4. Matching records are shown in the "Results Feedback" list.

Instructor Means Reporting

Measure instructor results against one another, or a benchmark group.

  1. Choose project(s).
  2. Choose hierarchy level(s).
  3. Choose instructor(s), or leave default settings (All Instructors).
  4. Choose course(s), or leave default settings (All Courses).
  5. Choose question on which to report.
    NOTE: A report may only contain questions which use the same scale (e.g., likert)
  6. Choose respondent or average criteria (e.g., where number of respondents greater than x, where average rating is greater than x)
  7. Choose to group by Course, Instructor, or Instructor/Course.
  8. Choose to aggregate question responses, or show reports for questions individually.
  9. Choose an area to serve as benchmark.
  10. Click Search.
  11. Matching records are shown in the "Results" list.
  12. Click any column header to sort the list by that criterion. Click the header again to reverse sort order.
  13. Click the appropriately labelled links to export results in PDF or Excel format.
