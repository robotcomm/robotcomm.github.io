Introduction‌

Identity Finder works like an anti-virus product. Client software installs locally, inspects storage media for data matching certain pre-defined criteria, and logs the activity with a management console operated by the Information Security Office. When pattern matches are found (e.g., potential social security numbers, credit card numbers) Identity Finder presents several options to remove the data:

  • Destroy the file (Shred)
  • Redact information from the file (Scrub)
  • Prepare the file for move to a secure location (Quarantine)
  • Safeguard the file using proprietary encryption (Secure)

In the case of a 'false positive' match users may also instruct Identity Finder to whitelist a file or location (Ignore).

Terms

Identity Type

Identity Types are the various ways in which a person may be identified to an institution or system. For example, a credit card allows you to buy something; a social security number allows you to sign up for a bank account; a bank account gives you access to your money; your mother's maiden name or your birth date is how you identify yourself on the phone to certain institutions.

Identity Finder searches for patterns that match identity types selected by Information Technology Services, Information Security Office.

End-users may select additional pre-defined identity types (may require additional configuration), or define custom search criteria using patterns (Regex, regular expressions), keywords and dictionary files (plain-text list).

OnlyFind

OnlyFind allows you to search for individual personal information. Identity Finder converts entered information into multiple formats to increase the likelihood of discovering matches. For example, a date of birth entered as 03/12/1972 will match any of the following formats in any location:

  • 03/12/1972
  • 03/12/72
  • 03-12-1972
  • 03-12-72
  • Mar. 12, 1972
  • March 12, 1972

Additionally, Identity Finder will also look for any words you enter in various cases (e.g., Maiden Name: Smith, smith, and SMITH - any one of these will match all three versions).

AnyFind

AnyFind is a proprietary technology that allows Identity Finder to search for selected Identity Types without requiring personal information. Identity Finder locates any data that resembles selected Identity Type.

Regex

A regular expression (abbreviated regex or regexp) is a sequence of characters that forms a search pattern, mainly for use in pattern matching with strings, or string matching. For example, the regular expression (?<=.) {2,}(?=[A-Z]) matches at least any occurrence of two spaces following a period (.) and preceding an upper case letter.

Dictionary file

Dictionaries are plain text files that use the following format: 


    [HEADER]
    Name=

    [WORDS]
    any words you want

Here is an example: 


    [HEADER]
    Name=DescriptiveName

    [WORDS]
    james
    john
    robert

NOTE: Dictionary words may only contain English alphabetic characters [A-Za-z], numbers [0-9], and periods [.]. Dictionary words are case insensitive.

Advanced scan

This process demonstrates the settings that can be used to run an advanced scan of a machine. It is not necessary for Case employees to use advanced scanning methods, as the software is pre-defined with the search settings required for the identification and elimination of restricted data.

  1. Open Identity Finder.
  2. Type your Identity Finder Profile password into the Enter Password field.
  3. Click OK.
  4. Select the Identities tab to customize search options.



  5. Click an / button to select it and activate with default settings.
  6. The Settings window appears; define values for any identity that uses OnlyFind. search. The Add Identity dropdown box allows you to select an identity to define.
  7. The value is added to the list of Only Find identity type values. Click the OK button to apply these settings. NOTE: Some identities may ONLY be used with OnlyFind. An OnlyFind value must be defined when an Identity Type is recognized, but does not necessarily follow a common pattern (e.g., passport number formats vary between issuing countries).
  8. Custom Type identities are defined by a user by either regular expression, keywords, or a dictionary file.
  9. The Locations tab allows users to define where the Identity Finder scans on a machine.
  10. Click the Configuration tab to define advanced application options and edit the appearance of scan results.
  11. Select the Locations tab to customize search locations.



  12. Click User Guide to see the vendor directions for using Identity Finder.

Scan Wizard‌

The Scan Wizard is an advanced option for users who wish to use more than the search options that have been programmed to default for use at Case Western Reserve University. It offers the same options as an advanced scan, but in a traditional Windows Wizard format for ease of use.

  1. Open Identity Finder.
  2. Type your Identity Finder Profile password into the Enter Password field.
  3. Click OK.
  4. Click the Start dropdown button to activate the menu.
  5. Select Start Search Wizard from the menu.
  6. The Identity Finder Wizard appears.
  7. Choose Identity Type(s) to search for.



  8. Click Next .
  9. If providing personal information for the OnlyFind search option choose Yes. Otherwise choose No.
  10. Click Next.
  11. Choose Locations where Identity Finder will search for restricted and sensitive data. NOTE: CWRU suggested setting is Files and Compressed Files, Browsers, and Emails and Attachments.
  12. Choose "My Documents and Settings" in the File Locations group box NOTE: Contact department or Division IT administrator before scanning networked file shares or server-side Home directories.



  13. Click Next.
  14. A Confirmation screen displays the Identities and Locations that Identity Finder will scan.



  15. Click Finish to begin scan or Back to change settings.
  16. Identity Finder begins searching your machine for sensitive data. You may continue with other tasks while the scan takes place. Search Results will appear when the scan is complete.
