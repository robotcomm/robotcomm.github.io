Service Description

The ITS Inventory process (Inventory) is the process by which ITS describes and catalogs its personal computing hardware resources.

Inventory requests are handled during regular business hours by Deskside Support Services (DSS).

Tools for Resolution

Required Information

  • Name,
  • CWRU Network ID,
  • Contact information (preferred and alternate),
  • Location (building name and room number)
  • Problem or Request description

Additional Resources

Assistance locating Serial Numbers, IMEI, etc.

Inventoried Items
The Inventory includes currently assigned personal computing hardware (desktops, laptops) within its useful lifecycle, docking stations, monitors and mobile devices (excluding cell phones). Newly acquired hardware is added to the inventory during the receiving process.

Personally owned equipment of any kind is not included in the Inventory. Software (regardless of license) is not included in the Inventory. Mobile phones, cabling, and peripherals / accessories (e.g., external hard drives, stands, wireless mice / keyboards) are also not inventoried.

Resolver Groups

DSS

How to Escalate

Description and purpose

An escalation may be requested by an end user when he or she determines that a critical issue has not been acknowledged properly, remains unresolved for an unacceptable period of time or is severe enough that it requires atypical handling.

Escalation should not be confused with assignment to a resolver group. An escalated issue is typically brought to the attention of management within a second-level support group(s).

Who to contact for escalation

Escalation available for procedural matters only.

Paul Kalanish; Paul.Kalanish@case.edu, 216.368.5911 (office)

Escalation procedures

ITS Inventory Process is not considered a critical service. Escalations are allowed during business hours.

Move + Change

Request a machine from inventory

return to index

  1. Create inventory ticket.
  2. Gather resolution information and enter it into the ticket.
  3. Verify that request is for inventoried item type.
  4. Verify that requester role is ITS Department Assistant (DA)
  5. Provide ticket number to requester.
  6. Forward to DSS resolver group.

NOTE: DA provides DSS detailed request form. No additional details are required to create ticket.

Send a machine to inventory

return to index

  1. Create inventory ticket.
  2. Gather resolution information and enter it into the ticket.
  3. Note that request type is “request for DSS to pickup a machine for service”.
  4. Schedule an appointment for DSS truck run.
  5. Provide ticket number to requester.
  6. Forward to DSS resolver group.

NOTE: User equipment with access to Internal Use Only data or Restricted data, and the Box Sync client may require special handling. Please note these cases in the ticket so that DSS may process appropriately.
