Service Description

Cisco TelePresence is a Learning Management System (LMS) used by instructors and students at CWRU. Cisco TelePresence may be used to supplement classroom-based courses, supply the online component of hybrid courses or deliver the content for distance education courses.

TermDefintion
   
Touchscreen Controller (Touch 8 or Touch 12)Touch-panel LCD device that enables you to conduct TelePresence meetings without the Cisco Unified IP phone.-
 
End Point (EP)The combination of hardware and software that comprise a Cisco TelePresence System. The hardware for an endpoint includes a touchscreen hardware controller, large-screen meeting display(s) with camera(s), presentation devices, microphones, speakers, and in some cases, lighting system.

NOTE: Some systems may also include a Cisco Unified IP phone.
   
Cisco WebEx-enabled callA call that has been configured to support Cisco WebEx online meeting capability including integrated phone conferencing, mobile access, voice conferencing, desktop sharing, and online recording.

Common Procedures

Instructions: Identify Cisco TelePresence hardware model
 
  1. Using the touchscreen controller, tap More > Settings
  2. Tap System Information on the left (this is shown by default)
  3. Scroll down to Hardware.
  4. Ask user to identify Product ID and Module Serial Number


Instructions: Restart Cisco TelePresence System
 
  1. Using the touchscreen controller, tap More > Settings
  2. Tap Restart on the left.
  3. Tap Restart in the center and then confirm by tapping Restart one more time.
  4. Wait for system to complete reboot (analyst should remain on phone with customer until system comes back online).

    NOTE: Restart processs may take as long as 5 minutes to complete.


Instructions: Enable/Disable Do Not Disturb (DND)
 

When set to Do Not Disturb ringtones are muted and call attempts made by others appear as missed calls.
Use the touchscreen controller to complete the steps below.

  1. To enable Do Not Disturb, tap Available (shown in upper left corner with green vertical bar).
  2. Tap Unavailable to select from the drop down menu..
  3. To disable Do Not Disturb, tap Unavailable (shown in upper left corner with red vertical bar).
  4. Tap Available to select from drop down menu.

Escalation Procedures


OVERVIEW  ^
 
Escalation procedures should be used when an issue is not being acknowledged or resolved in a manner that is acceptable to the end user. The end user will determine whether the issue is critical and may request an escalation. An escalation is not the same as assignment to a resolver group. Typically, escalations occur when it is necessary to bring an issue to the attention of management within a second level support group or groups.

 

PROCEDURE
 
All scheduled Cisco TelePresence events are supported. Cisco TelePresence general services are supported during weekday business hours (EST).

Troubleshoot / Repair   ^

I am unable to place a call.

Step

Activity
1. Gather resolution information and enter it into the ticket.
2. If caller is internal user (including John Hay), start with step 6. Otherwise, continue to step 3.
3. Ask if the user placed a support request at his or her home institution.
4. User HAS NOT requested support from home institution: Direct user to do so.
5.

Ask the user to attempt a call. Is the user able to initiate ANY call?

IF: User CAN place a call, BUT cannot connect to CWRU end point.
THEN: Note response and forward ticket to Cisco TelePresence resolver group.

IF: User CAN place a call AND connect to CWRU end point.
THEN: Resolve ticket.

IF: User is EXTERNAL and CANNOT place ANY call.
THEN: Instruct user to contact his or her home institution support group. Resolve ticket.

IF: User is OCH Partner and CANNOT place ANY call.
THEN: Troubleshoot beginning with step 6.

6. Walk user through process: Identify Cisco TelePresence model.
7. Ask user to verify End Point number. End point number should display on upper left or upper center of display device, and top of touchscreen controller.

Format example: 12163688889@case.edu
NOTE: If user cannot provide end point number user he or she may report an error message similar to one of the following;

No LAN
Not Registered
Network not available
8. Verify that physical connection (data) is firmly seated in both Cisco unit and wall jack.
9. Walk user through process: Restart Cisco TelePresence System.
10. OCH Partner: Resolve ticket AND refer user to his or her home institution support group.
Internal user: Resolve the ticket OR forward to the Cisco TelePresence resolver group.
