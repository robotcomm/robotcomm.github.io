Service Description

Blackboard is a Learning Management System (LMS) used by instructors and students at CWRU. Blackboard may be used to supplement classroom-based courses, supply the online component of hybrid courses or deliver the content for distance education courses.

Key features for more effective communication

  • Use blogs and wikis to facilitate ongoing writing assignments and collaborative projects.
  • Discussion boards enable moderated, rich, asynchronous conversations and drive student engagement.
  • Simplify periodic communications with automated email lists.

Key features for ease of use

  • Global navigation provides one-click access to frequently accessed destinations (including all applicable course sites.)
  • My Blackboard (access via the Global navigation menu) aggregates a user’s posts, grades, upcoming assignments and calendar entries.

Key features for rich content and media

  • Video everywhere allows any user to easily record from a webcam or mobile device and embed clips into the Content Editor.
  • The upgraded Content Editor is compatible with additional rich content and media types.
  • Use the new Content Collection feature to help ensure content freshness and consistent presentation across course sites.

Key features for instructors

  • Instructors may now use a Inline Assignment Grading to complete assignment review and grading seamlessly within Blackboard’s browser interface.
  • Monitor student progress and performance using the enhanced Retention Center feature.
  • Blackboard Learn is extended by Blackboard Wizard, an CWRU developed application that automates many manual tasks (e.g., setting up course sites, adding or removing users and migrating grades from Blackboard to SIS).
  • Deleted Site Recovery

Upgrade to Service Pack 14 (SP14)

Blackboard was upgraded to SP14 from SP11 in May, 2014. Several small changes to the Blackboard product include:

  1. Additional test features, including exceptions for individual students and new feedback options.
  2. A test access log so instructors can see how much time a student spent on their test attempt.
  3. Improved functionality for managing groups within Blackboard courses.
  4. Date management tool that allows instructors to make bulk changes to dates within their course site.
  5. Achievements, Blackboard’s badging tool that allows instructors to gamify their lessons.
  6. Improvements to the Grade Center look-and-feel.
  7. A new look-and-feel to the Instructor UI for blogs, wikis, and journals.

The largest change is the enabling of Notifications. Notifications allow users to receive email and mobile notifications based on updates in their Blackboard courses.

How to Escalate

Description and purpose

An escalation may be requested by an end user when he or she determines that a critical issue has not been acknowledged properly, remains unresolved for an unacceptable period of time or is severe enough that it requires atypical handling.

Escalation should not be confused with assignment to a resolver group. An escalated issue is typically brought to the attention of management within a second-level support group(s).

Blackboard is considered a critical service.

During unplanned university closures escalations may NOT be available.

Detailed Troubleshoot / Repair Procedures

Student cannot access his or her course

return to index

  1. Gather resolution information and enter it into the ticket.
  2. Sign in to Blackboard and select the System Admin tab.

  3. Click Courses to search by course name, ID or instructor.

    Format search parameters as shown:
    Course name: ‘CHEM 101’
    Course ID: ‘CHEM101_abc123’
    Instructor: ‘abc123’

  4. Course NOT FOUND, Instructor has not added the course to Blackboard. Refer student to Instructor.
  5. Course FOUND, Review status column.
  6. Status NOT AVAILABLE, Course not yet made available to students by instructor.
    Forward ticket to Blackboard resolver group with request to contact
    instructor w/r to date of availability.
  7. Status AVAILABLE, Enter details into ticket and forward to Blackboard resolver group.
  8. Resolve the ticket.

NOTE: NOTE: If you need access to either Blackboard or the System Admin
tab send a message to kas200@case.edu to request
permission.

Instructor cannot find his or her course

return to index

  1. Gather resolution information and enter it into the ticket.
  2. Course HAS NOT been hosted by instructor in Blackboard prior OR,
  3. Course HAS been hosted by instructor in Blackboard prior AND content WILL NOT be reused
  4. Walk the instructor through process of creating a new course and making it available.
  5. Create a new course
    1. Log in to Blackboard.
    2. Click the Blackboard Wizard tab located at the top of the Blackboard page.
    3. Click Add a New Blackboard site on the left menu.
    4. Choose Semester and click Next.
    5. Choose the Course and click Next.
    6. Enter in the Site Title and click Next.
    7. Create the Course Site ID and click Next.
    8. A confirmation page will appear. Review information and click Create site.
  6. Make the site available
    1. Log in to Blackboard.
    2. Click the desired course from My Courses or the Global Navigation list.
    3. From the Control Panel, select Customization.
    4. Select Properties.
    5. Under set Availability, select Yes to make the course available. If you do not want to make the site available, select No.
    6. Click Submit.
    7. Refer also to the Blackboard Wizard guide.
  7. Course in question HAS been hosted by instructor in Blackboard prior, AND instructor WILL reuse course content,
  8. Course Copy
    1. Verify that a new course site has been created. Refer to the Blackboard Wizard guide.
    2. Sign in to Blackboard.
    3. Navigate to original (source) content site.
    4. Select Packages and Utilities > Course Copy from the Control Panel menu.
    5. Click Browse to find the new (destination) course.
    6. Select the check box next to the new (destination) course name and then click Submit.
    7. Select check boxes for content to copy.
    8. Do NOT select Include Enrollments in the Copy.
    9. To copy Blackboard assignments, select Grade Center Column > Settings AND Content Area check boxes where the assignments are stored.
    10. Click Submit.
  9. Instructor WILL reuse a course site,
    1. Refer user to steps for #6 “Instructor WILL reuse course content…”
    2. Faculty support in ITS encourages instructors to create new course sites each semester.
  10. Note all troubleshooting steps taken.
  11. Resolve the ticket.

NOTE: User will receive a confirmation at his or her CWRU email address
when process is complete. Resubmitting before the receipt of process
completion may result in duplicate materials and should be avoided.

Course Site ID CANNOT be changed. If you plan to reuse the course,
avoid any unique identifiers.

One or more pages are not loading properly

return to index

  1. Gather resolution information and enter it into the ticket.
  2. Verify internet connectivity.
  3. Verify user is able to sign in to Blackboard using CWRU ID and password.
  4. Verify user’s browser is supported. Refer to Browser Support for Service Pack (SP) 11.
  5. Instruct user to log out of Blackboard, clear their cache/cookies and retry.
  6. Record the name of pages that are malfunctioning, if specific slow response and enter this information into the ticket.
  7. Note all troubleshooting steps taken.
  8. Resolve the ticket or forward to Blackboard resolver group.

NOTE: Blackboard support strongly recommends the Firefox, Chrome and
Opera web browsers for the best Blackboard experience. Blackboard support
recommends that users avoid both Internet Explorer and Safari.

    1. Sign in to Blackboard.
    2. Click System Admin tab.
    3. Click Courses and search for the relevant course.

    1. Access the Retention Center.
    2. Hover over the Create Rule button and click rule-type option.
    3. Name the rule and decide whether to include it in the at-risk table.
    4. Define rule criteria.
    5. Click Submit.

More about Notifications

return to index

  1. Gather resolution information and enter into the ticket.
  2. Notifications provide email and mobile updates to users based on changes made in their Blackboard sites.
  3. Notifications can be customized to the user’s preferences.
  4. Have user login to Blackboard and click their name in the upper right corner of the screen.
  5. Select the Updates page (clock icon).
  6. Click the gear icon in the upper right corner. This will allow users to choose what events trigger a notification.
  7. Click Save.
  8. To change the frequency of received notification emails, have the user follow the above steps 1-6, then click View Notification Settings.
  9. Click Edit General Settings and change notification preferences.
  10. Click Submit to save changes.
  11. Note all procedure steps taken.
  12. Resolve the ticket.

More about Mobile Notifications

return to index

  1. Gather resolution information and enter into the ticket.
  2. The user must have the Blackboard Mobile Learn application installed on their mobile device for mobile notifications to work.
  3. Have user login to Blackboard and click their name in the upper right corner of the screen.
  4. Select the Updates page (clock icon).
  5. Click the gear icon in the upper right corner.
  6. Click View Notification Settings.
  7. If the user wants to receive mobile notifications for all courses, have them select Courses I am taking under Bulk Edit Notification Settings. Otherwise, have them select the individual course they want to make changes to.
  8. Select the checkboxes in the Mobile column next to the items that should generate a mobile notification.
  9. Click Submit to save changes.
  10. Let the user know that it may a small amount of time before notifications begin to appear on their device (less than an hour).
  11. Let the user know that they will need notifications for the Mobile Learn app enabled on their device for push notifications to appear.
  12. Note all procedure steps taken.
  13. Resolve the ticket.

More about Date Management

return to index

  1. Gather resolution information and enter into the ticket.
  2. The date management tool allows instructors to adjust due dates, adaptive release dates, and availability dates for all dates within their Blackboard site. This is helpful after the course copy process or when recycling content from one semester to the next.
  3. Enter into the site that needs dates adjusted.
  4. From the Control Panel, select Course Tools then Date Management.
  5. Select how to adjust the dates then click Start.
  6. The tool will run and will display a load bar. Once complete, click Next.
  7. A review page will appear. To make further edits, click the Pencil Icon next to each item.
  8. Note all procedure steps taken.
  9. Resolve the ticket.
