Service Description

Rack of videoconferencing equipment located in the Cedar Avenue Service Center (CASC) allows users and analysts to perform test calls prior to scheduled videoconference events.

Tools for Resolution

Required Information

  • Name
  • CWRU Network ID
  • Contact information (preferred and alternate)
  • Location (building name and room number)
  • Department or division
  • Description of equipment
  • Date of scheduled conference call
  • Description of problem or request

Resolver Groups

Videoconferencing: Available Mon.-Fri., 8 a.m - 5 p.m., EST (and evenings as scheduled). Responses to support requests are provided within two working days.

Urgent requests (e.g., scheduled event less than two working days from request) will be attended as quickly as possible. However, resolution prior to the date scheduled event cannot be guaranteed.

How to escalate

Description and purpose

An escalation may be requested by an end user when he or she determines that a critical issue has not been acknowledged properly, remains unresolved for an unacceptable period of time or is severe enough that it requires atypical handling.

Escalation should not be confused with assignment to a resolver group. An escalated issue is typically brought to the attention of management within a second-level support group(s).

Escalation procedures

Videoconference test calling is not considered a critical service. Escalations are not available at this time. All unresolved issues should be assigned to the

Videoconferencing resolver group. Support requests are classified as P4 without regard to urgency. Outages and potential outages are classified as P2 events.

Troubleshoot + Repair

Choose the right test endpoint or bridge

back to top

  1. Gather resolution information and enter it into the ticket.
  2. Determine equipment type
  3. Does the user have Polycom, Cisco Tandberg or Lifesize branded equipment?
  4. IF YES - this is Internet Protocol (IP) equipment.

  5. Choose a test procedure based on the location of user and conference participants.

    Calling outside campus, from campus
    test using 192.5.109.6

    Calling to campus, from outside
    test using 129.22.157.145

    Cleveland Clinic Classroom participants
    test using 192.5.109.51

  6. IF NO - test using Telepresence.

  7. Test with standard bridge and VNOC endpoint, unless user is affiliated with Cleveland Clinic Classroom.

    Any caller

    1. 12167542100@case.edu (bridge)
    2. 12167541410@case.edu (physical endpoint)

    Cleveland Clinic Classroom participants
    test using 12167542514@case.edu

  8. Complete test call using provided procedures.
  9. Resolve ticket if successful.

Perform a test call using IP teleconference equipment

back to top

  1. Gather resolution information and enter it into the ticket.
  2. User must be able to access teleconference equipment to perform test call.
  3. If user does not have immediate access to teleconference equipment, schedule callback and provide user ticket number for reference.
  4. Instruct user to call the appropriate teleconference endpoint (see, Choose the right test endpoint or bridge) from his or her teleconference equipment.
  5. Verify the following and note user response in ticket -
    1. User can see video.
    2. User can hear audio.
    3. Green audio meter on left is visible, and moving (user audio levels)
    4. Green audio meter on right is visible, and moving (VNOC audio levels)
  6. If any of these tests fail, note the point of failure in the ticket and assign to Videoconferencing resolver group.
  7. Describe any other failure (e.g., ring heard, but endpoint doesn’t answer, etc.).
  8. Resolve ticket.

Perform a test call using Cisco Telepresence equipment

back to top

  1. Gather resolution information and enter it into the ticket.
  2. User must be able to access teleconference equipment to perform test call.
  3. If user does not have immediate access to telepresence equipment, schedule callback and provide user ticket number for reference.
  4. Perform a test using the Case Test Bridge
  5. Instruct user to call 12167542100@case.edu from his or her telepresence unit.
  6. When “Case Test Bridge” message appears in video window, user has connected successfully.
  7. Analyst should then call 12167542100@case.edu from his or her Jabber client.
  8. When user and analyst can see and hear one another, test is completed.
  9. If any point of failure, note these and attempt to complete a test using the VNOC endpoint.
  10. Perform a test using the VNOC endpoint
  11. Instruct user to call 12167541410@case.edu from his or her telepresence unit.
  12. Verify the following and note user response in ticket -
    1. User can see video.
    2. User can hear audio.
    3. Green audio meter on left is visible, and moving (user audio levels)
    4. Green audio meter on right is visible, and moving (VNOC audio levels)
  13. If any of these tests fail, note the point of failure in the ticket and assign to Videoconferencing resolver group.
  14. Describe any other failure (e.g., ring heard, but endpoint doesn’t answer, etc.).
  15. Resolve ticket.

Prerequisites for test calling from computer

back to top

  1. Gather resolution information and enter it into the ticket.
  2. To make a Jabber call from a computer the following are prerequisite requirements,
    • An integrated or external web camera
    • An integrated or external microphone
    • Installed Jabber client and Jabber account
  3. Resolve ticket.

Install Jabber

back to top

  1. Gather resolution information and enter it into the ticket.

  2. Refer to the procedures below. NOTE: Jabber Quick Start Guide available in PDF format from Mediavision.

    Download and install the client

    1. Go to ciscojabbervideo.com.
    2. Click the green “Get Jabber Video” button.
    3. Fill out the form completely and click Sign Up.
    4. The software installer will download automatically.
    5. Choose “Run” to open the installer.
    6. Follow installer instructions to complete the software installation.

    Setup and test the client

    1. Open Jabber application. User will be guided through client setup, sound and video test.
    2. Type 12167542100@case.edu into the field marked, “Type name, number or address”.
    3. Press Enter.
    4. When “Case Test Bridge” message appears in video window, user has connected successfully.

    NOTE: if calling from off-campus, start Cisco VPN client before launching Jabber.

    Call in to a videoconference

    1. Open Jabber application.
    2. Type the meeting telephone number (provided by host or organizer) into the field marked, “Type name, number or address”.
    3. Press Enter.
    4. Videoconference begins.
  3. Resolve the ticket.
