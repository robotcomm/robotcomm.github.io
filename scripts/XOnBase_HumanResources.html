Service Description

OnBase is an Enterprise Document Management (EDM) system used by Human Resources.

Tools for Resolution

Required Information

  • Name,
  • CWRU Network ID,
  • Hostname,
  • Contact information (preferred and alternate),
  • Department,
  • Location (building name, room number),
  • Error message(s) (if applicable),
  • Problem or Request description

Additional Resources

  • Client type: Web-based (thin). Legacy thick client is not supported.
  • Supported browser(s): IE8-IE10
  • Access details: CWRU Network ID and password
  • Application URL: edocm.case.edu
  • Server location: ldb-ksl-3
  • Departments served with application instance: Human Resources

Resolver Groups

OnBase HR, EDocM

How to Escalate

Description and purpose

An escalation may be requested by an end user when he or she determines that a critical issue has not been acknowledged properly, remains unresolved for an unacceptable period of time or is severe enough that it requires atypical handling.

Escalation should not be confused with assignment to a resolver group. An escalated issue is typically brought to the attention of management within a second-level support group(s).

Who to contact for escalation

Kelli Caponero, kxc248@case.edu, (216) 368-8970 (office)
Josh Kendall, josh.kendall@case.edu, (216) 368-0401 (office)

Escalation procedures

Document Management is not considered a critical service.

Escalation is available Mon.-Fri., 8 a.m - 5 p.m., EST. Escalations are are not available during or unplanned university closures, or during recognized university holidays.

Classify the reason for escalation as one of the following; Confirmed Outage (total or partial), Suspected Outage (total or partial), Abend, Slow response time, User Requirements not met, Other functional or Other technical.

Detailed Troubleshoot / Repair Procedures

return to index

User needs help connecting to OnBase

return to index

  1. Gather resolution information and enter it into the ticket.
  2. Verify browser is supported and enter browser version in ticket. (IE8, IE9 and IE10 are supported)
  3. Verify user is entering correct URL.
  4. Instruct user to attempt with correct URL and supported browser.
  5. Troubleshoot wired and wireless connectivity as appropriate.
  6. Can user access the internet?
  7. User cannot access internet in browser,
    • Run anti-virus
    • Reboot machine
    • Retry
  8. User can access internet, but not case.edu site(s),
    • Clear browser cache
    • Clear cookies
    • Reboot machine
    • Retry
  9. Note any troubleshooting steps taken.
  10. Verify that server (ldb-ksl-3) is up.
  11. If server is available and you still cannot access OnBase treat the incident as a potential outage.
  12. If server is unavailable advise user that (1) OnBase is temporarily unavailable due to an outage AND (2) ITS is working to resolve the issue.
  13. Provide an ETA for resolution ONLY when given in Footprints global ticket.
  14. Resolve the ticket or forward to OnBase HR resolver group.

NOTE: Off-campus users with thick-client must connect with VPN. This is NOT COMMON.

OnBase is not responding

return to index

  1. Gather resolution information and enter it into the ticket.
  2. Troubleshoot wired and wireless connectivity as appropriate.
  3. If the issue is impacts OnBase only, troubleshoot using the Connecting to OnBase instructions.
  4. If the issue is not specific to OnBase, troubleshoot as appropriate.
  5. Include the scope of the issue in the ticket, if it can be determined.
  6. Note any troubleshooting steps taken.
  7. Resolve the ticket or forward to OnBase HR resolver group.

NOTE: If user is a member of OnBase HR resolver group, ticket should be assigned to EDocM.

Detailed Outage Procedures

return to index

Outage topic.

return to index

  1. Gather resolution information and enter it into the ticket.
  2. Troubleshoot using the Connecting to OnBase instructions.
  3. Verify that server (ldb-ksl-3) is up.
  4. If server is available and you still cannot access OnBase, treat the incident as a potential outage.
  5. Forward to OnBase HR resolver group. Or escalate, if requested.
  6. If server is unavailable advise user that (1) OnBase is temporarily unavailable due to an outage AND (2) ITS is working to resolve the issue.
  7. Follow Service Interruption Communication Procedure scripts.
  8. Provide an ETA for resolution ONLY when given in Footprints global ticket.
  9. Forward to OnBase HR resolver group. Or escalate, if requested.

NOTE: If user is a member of OnBase HR resolver group, ticket should be assigned to EDocM.
