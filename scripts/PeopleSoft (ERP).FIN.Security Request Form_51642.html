PeopleSoft | Financials | Security Request Form

Keywords

PeopleSoft; Financials; FIN; ERP; Access; Security access; Security request; Security request form; Self-service request

Service Description

By default, ERP Security provides users suffcient privilege to submit a request for access to secured Financials (ERP FIN) modules.

Access may be requested for the following modules in any combination;

  • eProcurement
  • Accounts Payable
  • General Ledger
  • Reporting
  • Petty Cash

When completing a security request form (available online only), the user will be required to provide;

  • Requester name and contact information
  • Requester’s supervisor/approver name and contact information

NOTE: Affiliates (e.g., Kelly Services et. al) must be added to PeopleSoft Financials manually by ERP Security.

Tools for Resolution

Required Information

  • Name,
  • CWRU Network ID,
  • Contact information (preferred and alternate),
  • Problem or Request description,
  • User status (new user, or existing user requesting different access),
  • Name of form submitter (if different from caller)
  • Form completion status (complete, incomplete, prevented from completion - technical, complete with errors, etc.)
  • School or Management Center (6 digit dept. code, Ex: 231231)
  • Name of supervisor
  • Name of requisition approver (if different from supervisor)

How to Escalate

Description and purpose

An escalation may be requested by an end user when he or she determines that a critical issue has not been acknowledged properly, remains unresolved for an unacceptable period of time or is severe enough that it requires atypical handling.

Escalation should not be confused with assignment to a resolver group. An escalated issue is typically brought to the attention of management within a second-level support group(s).

Escalation procedures

The ERP (FIN) Security Request form is not considered a critical service.

Escalation is available Mon.-Fri., 8 a.m - 5 p.m., EST. Escalations are are not available during or unplanned university closures, or during recognized university holidays.

Classify the reason for escalation as one of the following; Confirmed Outage (total or partial), Suspected Outage (total or partial), Abend, Slow response time, User Requirements not met, Other functional or Other technical.

User requested access to security form, but is unable to sign in.

return to index

  1. Gather resolution information and enter it into the ticket.
  2. When the security form is submitted, both the person who completed the form and the person requesting access will receive three emails:
    • After the form is submitted
    • After training has been completed and entered by the training team
    • After access is approved
  3. User has not received ANY email:
  4. Assign the Footprints ticket to the ERP Security resolver group.
  5. User has received initial email ONLY:
  6. User information is being routed (user will receive notification when access is granted).
  7. User has received all emails, AND completed training:
  8. Observe user (Bomgar) while attempting to sign in.
  9. Verify user is unable to access FIN module for which they have requested access.
  10. Note all troubleshooting steps taken.
  11. Resolve the ticket or forward to ERP Security resolver group.

Access request process

return to index

  1. Gather resolution information and enter it into the ticket.
  2. ERP Security provides access to a form that must be completed for each user requesting access to a secured FIN module.
  3. This form may be completed by the person requesting access, or his or her supervisor.
  4. When the security form is submitted, both the person who completed the form and the person requesting access will receive three emails:
    • After the form is submitted
    • After training has been completed and entered by the training team
    • After access is approved

    1. Note all troubleshooting steps taken.
  6. Resolve the ticket or forward to ERP Security resolver group.

NOTE: Users will be unable to access Financials until training is complete (E-Procurement (aka e-pro or purchasing), AP (Accounts Payable), GL (General Ledger). Affiliates (i.e. Kelly Services et. al) must be added to PeopleSoft Financials manually by ERP Security.
