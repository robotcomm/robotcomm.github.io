Keywords

Open SSL; Vulnerability; Security; Heartbleed; HTTPS; Data theft

Service Description

The security vulnerability "Heartbleed" puts at risk all user information traversing the Internet between user computers and servers that use Open SSL. Web sites that use Open SSL are differentiated by the prefix "HTTPS".

"Heartbleed" permits password and ID theft. Any information shared between the user and the server that would normally be protected during the session is vulnerable.

NOTE: More technical information can be found in the Qualys SSL Testing site.

What actions is CWRU taking to safeguard systems against "Heartbleed"?

return to training questions

  1. Gather resolution information and enter it into the ticket.
  2. ITS and university IT administrators have confirmed that none of the core critical IT services using SSL, including Single Sign-On or www.case.edu, are affected by this vulnerability.
  3. Additional analysis of CWRU servers and systems is ongoing.
  4. CWRU users are advised to exercise caution with websites they visit. The exploit can affect both servers and users’ Web browsers.
  5. Forward to Information Security resolver group.

NOTE: Although there has been no evidence that a CWRU website has been compromised, we now know that the vulnerability has existed since March 2012.

What actions can I take to personal my personal data against theft?

return to training questions

  1. Gather resolution information and enter it into the ticket.
  2. Test sites they use with Heartbleed Test Site.
  3. In addition, users can ensure the security of their Web browsers and plugins by using the CWRU Browser Check plugin.
  4. We urge CWRU users to subscribe to multifactor authentication with popular services, such as Google, Yahoo, Facebook, iCloud, Evernote and Twitter.
  5. CWRU is in the testing phase of its own multifactor authentication service, and interested users are invited to test the service.
  6. CWRU users are advised to exercise caution with websites they visit. The exploit can affect both servers and users’ Web browsers.
  7. We anticipate a new wave of phishing messages using this vulnerability as an excuse to steal login credentials and compromise accounts. Beware of spam messages.
  8. Forward to Information Security resolver group.

What internet browswers are preferred for security against "Heartbleed"?

return to training questions

  1. Gather resolution information and enter it into the ticket.
  2. Users can expect all major browsers to address this issue with an update (e.g. Chrome has already released an update).
  3. Forward to Information Security resolver group.

I have received an email message requesting I change login credentials. What should I do?

return to training questions

  1. Gather resolution information and enter it into the ticket.
  2. At this time, CWRU is not asking users to change their CWRU network passwords.
  3. Server and site owners may notify users to change their passwords with the service providers as servers are patched.
  4. We anticipate a new wave of phishing messages using this vulnerability as an excuse to steal login credentials and compromise accounts. Beware of spam messages.
  5. Forward to Information Security resolver group.
