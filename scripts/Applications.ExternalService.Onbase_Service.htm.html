Service Description

OnBase is an Enterprise Document Management (EDM) system used by Enrollment Management, Financial Aid and HR.
Configuration and support details vary by department.

  • Client type; Web-based (thin). Legacy thick client is not supported.
  • Supported browser(s); IE8-IE10
  • Access details; CWRU Network ID and password
  • Server location; ldb-ksl-3
  • Departments served; Enrollment Management (Enrollment Management, Financial Aid, Student Employment, Undergraduate Admissions, Alumni Affairs, Enrollment Operations), Human Resources, Accounts Payable Invoices (Accounts Payable Invoices, Campus Services, Procurement, Construction, Facilities)
  • Support group (Business hours); Enrollment Management - ITEM, Human Resources - OnBase HR, Accounts Payable Invoices - OnBase AP
  • Support group (after hours); Enrollment Management, Human Resources, Accounts Payable Invoices - Service Desk

NOTE: An internal user is a CWRU staff person who is a self-identified member of any of the departments listed under ‘departments served.’

Use this article when:

  1. An internal user has any question about OnBase implementation or procedures.
  2. Any user requests OnBase access or setup for his or her department (a new user group).
  3. Any user reports an issue with OnBase OR reports a suspected outage of OnBase.

Escalation

An escalation may be requested by an end user when he or she determines that a critical issue has not been acknowledged properly, remains unresolved for an unacceptable period of time or is severe enough that it requires atypical handling.

An escalated issue is typically brought to the attention of management within a second-level support group(s).

Escalation should not be confused with assignment to a resolver group.

Detailed Troubleshoot / Repair Procedures

return to top | index | troubleshoot questions

I need help connecting to OnBase.

return to troubleshoot questions

  1. Gather resolution information and enter it into the ticket.
  2. Verify that ldb-ksl-3 is up.
    1. Server is UP
    2. Attempt to access OnBase
    3. If unable to access, treat this as a possible outage.
    4. Server is DOWN
    5. Advise user (1) OnBase is temporarily unavailable due to an outage AND (2) ITS is working to resolve the issue.
    6. Provide an ETA for resolution ONLY if provided in Footprints global ticket.
  3. Verify browser is supported and enter browser version in ticket. IE8, IE9 and IE10 are supported.
  4. Verify user is entering correct URL.
  5. Instruct user to attempt with correct URL and supported browser.
  6. IF off-campus location AND using legacy thick-client, THEN user MUST connect with VPN. This is NOT COMMON.
  7. Troubleshoot wired and wireless connectivity as appropriate.
  8. IF NEITHER external website(s) NOR case.edu site(s) are accessible, THEN
    1. Run anti-virus
    2. Reboot machine
    3. Retry
  9. IF external website(s) ARE ACCESSIBLE, BUT case.edu site(s) are not, THEN
    1. Clear browser cache
    2. Clear cookies
    3. Reboot machine
    4. Retry
  10. IF BOTH external website(s) AND case.edu site(s) ARE ACCESSIBLE, THEN
    1. Resolve the ticket.
  11. Note any troubleshooting steps taken in the ticket.
  12. If issue is limited to OnBase and unresolved, forward to resolver group.

NOTE:

  • Human Resources, forward to OnBase HR resolver group.
  • Accounts Payable Invoices, forward to OnBase AP resolver group.
  • Enrollment Management or Other, forward to ITEM resolver group.

The OnBase application is not responding.

return to troubleshoot questions

  1. Gather resolution information and enter it into the ticket.
  2. Troubleshoot wired and wireless connectivity as appropriate.
  3. Determine if issue is limited to OnBase.
  4. OnBase issue only
    1. Troubleshoot per Connecting to OnBase steps.
  5. Generalized issue
    1. Troubleshoot issues as appropriate.
    2. Follow resolution or escalation steps as provided for specific issue, if the issue can be determined.
  6. Note any troubleshooting steps taken in ticket.
  7. If issue is limited to OnBase and unresolved, forward to resolver group.

NOTE:

  • Human Resources, forward to OnBase HR resolver group.
  • Accounts Payable Invoices, forward to OnBase AP resolver group.
  • Enrollment Management or Other, forward to ITEM resolver group.

The ad hoc button is not responding (workflow function).

return to troubleshoot questions

  1. Gather resolution information and enter it into the ticket.
  2. Determine user department.
    1. Human Resources, resolve ticket. HR does not use workflows.
    2. Accounts Payable Invoices, troubleshoot per Connecting to OnBase steps.
    3. Enrollment Management, troubleshoot per Connecting to OnBase steps.
  3. Note any troubleshooting steps taken in ticket.
  4. If issue is limited to OnBase and unresolved, forward to resolver group.

NOTE:

  • Human Resources, forward to OnBase HR resolver group.
  • Accounts Payable Invoices, forward to OnBase AP resolver group.
  • Enrollment Management or Other, forward to ITEM resolver group.
