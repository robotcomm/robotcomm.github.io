Service Description

This document describes the initial information gathering tasks needed for storage and server space requests. These requests will be forwarded to departmental Share Admin or ITS resolver group, as appropriate for processing.

  • CWRU Network ID,
  • Contact information (preferred and alternate),
  • Problem or Request description
  • Department
  • Share Admin

Escalation

An escalation may be requested by an end user when he or she determines that a critical issue has not been acknowledged properly, remains unresolved for an unacceptable period of time or is severe enough that it requires atypical handling.

An escalated issue is typically brought to the attention of management within a second-level support group(s).

Escalation should not be confused with assignment to a resolver group.

Detailed Troubleshoot / Repair Procedures

return to top | index | troubleshoot questions

I received a low storage space warning and need to request additional storage space.

return to troubleshoot questions

  1. Gather resolution information and enter it into the ticket.
  2. Type Share Admin name into the ticket.
  3. Provide the Share Admin name to the user.
  4. Ask if user would like assistance freeing storage space by removing old, unused or redundant files.
  5. Enter response into ticket.
  6. Advise user of ITS Backup Service Level Agreement (http://www.case.edu/its/backup/).
  7. Instruct user to contact Primary Share Admin. If Primary Share Admin is either DSS or Servers and Storage Engineering, then assign to that resolver group.
  8. If user is Primary Share Admin, then assign ticket to Secondary Share Admin resolver group. If Secondary Share Admin is not listed, then assign to Servers and Storage Engineering resolver group.
  9. Resolve the ticket.

I cannot connect to a mapped network drive / share.

return to troubleshoot questions

  1. Gather resolution information and enter it into the ticket.
  2. Advise user that ticket will be handled by Deskside Support.
  3. Forward ticket to DSS resolver group.

I cannot (edit/delete/view) files on a mapped network drive / share when I should be able to, but can connect.

return to troubleshoot questions

  1. Gather resolution information and enter it into the ticket.
  2. Advise user that ticket will be handled by Deskside Support.
  3. Forward ticket to DSS resolver group.

return to top | index | setup questions

I need to request additional storage / server space (no specified purpose).

return to setup questions

  1. Gather resolution information and enter it into the ticket.
  2. Type Share Admin name into the ticket.
  3. Provide the Share Admin name to the user.
  4. Ask user if request is due to a low storage space warning.
  5. Ask if user would like assistance freeing storage space by removing old, unused or redundant files.
  6. Enter response into ticket.
  7. Advise user of ITS Backup Service Level Agreement (http://www.case.edu/its/backup/).
  8. Instruct user to contact Primary Share Admin. If Primary Share Admin is either DSS or Servers and Storage Engineering, then assign to that resolver group.
  9. If user is Primary Share Admin, then assign ticket to Secondary Share Admin resolver group. If Secondary Share Admin is not listed, then assign to Servers and Storage Engineering resolver group.
  10. Resolve the ticket.

I need to request additional storage / server space (specified purpose).

return to setup questions

  1. Gather resolution information and enter it into the ticket.
  2. Type Share Admin name into the ticket.
  3. Provide the Share Admin name to the user.
  4. Ask user to briefly describe the intended use (e.g., application name, storage, redundant support).
  5. Enter response into ticket.
  6. Instruct user to contact Primary Share Admin. If Primary Share Admin is either DSS or Servers and Storage Engineering, then assign to that resolver group.
  7. If user is Primary Share Admin, then assign ticket to Secondary Share Admin resolver group. If Secondary Share Admin is not listed, then assign to Servers and Storage Engineering resolver group.
  8. Resolve the ticket.

I need to request storage / server space for the first time.

return to setup questions

  1. Gather resolution information and enter it into the ticket.
  2. Type Share Admin name into the ticket.
  3. Provide the Share Admin name to the user.
  4. Ask user to briefly describe the intended use (e.g., application name, storage, redundant support).
  5. Ask user to briefly describe additional requirements the Share Admin should consider (e.g., documentation, training, communication or consulting).
  6. Enter responses into ticket.
  7. Instruct user to contact Primary Share Admin. If Primary Share Admin is either DSS or Servers and Storage Engineering, then assign to that resolver group.
  8. If user is Primary Share Admin, then assign ticket to Secondary Share Admin resolver group. If Secondary Share Admin is not listed, then assign to Servers and Storage Engineering resolver group.
  9. Resolve the ticket.
