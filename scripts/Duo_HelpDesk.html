Service Description

Two-factor authentication enhances the security of user logins when accessing CWRU resources by employing a phone or other connected mobile device as a second method of verifying user identity. 

Authentication is provided by the vendor, DuoSecurity. ITS has integrated DuoSecurity with VPN service. More services are expected to be integrated with vendor-provided authentication throughout FY15.



Use this article when:

  1. Any user with valid credentials needs assistance,
    • enrolling devices
    • managing devices
    • authenticating via Duo using any named method
  2. Any user with valid credentials needs assistance identifying a supported VPN profile
  3. Any enrolled user wishes to direct a comment to the service owner, request features or file a complaint.
  4. Any user has a general question about implementation of DuoSecurity; including program details, roadmaps for future integration(s), program participation eligibility or policy questions.
  5. Any user needs to report a service outage


NOTE: General questions about VPN, VPN troubleshooting or the Cisco AnyConnect client are outside of the scope of these instructions. Please see topic-specific instructions.

Description and purpose

An escalation may be requested by an end user when he or she determines that a critical issue has not been acknowledged properly, remains unresolved for an unacceptable period of time or is severe enough that it requires atypical handling.

Escalation should not be confused with assignment to a resolver group. An escalated issue is typically brought to the attention of management within a second-level support group(s).

Escalation procedures

DuoSecurity two-factor authentication and coordinating VPN profile support is not considered a critical service.

Escalation is available Mon.-Fri., 8 a.m - 5 p.m., EST. Escalations are are not available during planned or unplanned university closures, or during recognized university holidays.

Classify the reason for escalation as one of the following; Confirmed Outage (total or partial), Suspected Outage (total or partial), Abend, Slow response time, User Requirements not met, Other functional or Other technical.

Troubleshoot alternative authentication

return to index

  1. Gather resolution information and enter it into the ticket.
  2. ITS and Information Security recommend that all users authenticate via the mobile application using the passcode method.
  3. Determine method of authentication that is failing; voice call, text (SMS) or DuoPush.
  4. If DuoPush, note mobile device type (Android, iOS or Other).
  5. Verify that has network connectivity to their mobile device (cellular or WiFi).
  6. Verify that user has access to smartphone, has installed the application and completed the self-enrollment process.
  7. Guide the user through self-enrollment process if necessary.
  8. Clear browser cache, close browser and retry.
  9. Refer to method-specific directions following.
  10. Try additional authentication methods, starting with the preferred "passcode" feature.
  11. Instruct user to access CWRU VPN using non-Duo VPN profile until the issue is resolved.
  12. Document troubleshooting steps taken in ticket.
  13. Add solutions / troubleshooting steps / links to external resources not included with instructions to ticket.
  14. Resolve ticket or assign to Information Security resolver group.


Authentication via voice call failed

  1. Repeat voice authentication process
  2. User did not receive authentication phone call;
    • Can user receive any call at specified phone number?
    • Are calls received at specified phone number forwarded to another extension or outside line?
    • Is handset properly daisy-chained or connected to faceplate correctly?
    • In the enrollement page, the user must select the phone on which they want to have the system call them.
    • In the VPN, they should enter "phone" into the Second Password field to have the system call their primary device, or "phone2" to call their secondary device, etc.
    • Note point of failure in ticket: hardware.
  3. User received authentication phone call; authentication successful;
    • Note point of failure in ticket: user error.
  4. User received authentication phone call; authentication failed;
    • Note point of failure in ticket: authentication failed: [method]


Authentication via text (SMS) failed

  1. Repeat SMS authentication process
  2. User did not receive SMS;
    • Users cannot receive SMS messages on CWRU VoIP land line phones.
    • Does the user's cell phone plan include SMS messages?
    • Charges for messages received are the responsibility of receiver.
    • Does the user filter SMS messages in any way (e.g., using an application like Google Talk) or using a hardware / OS function? Troubleshoot as appropriate.
    • Note point of failure in ticket.
  3. User received SMS; authentication successful;
    • Note point of failure in ticket: user error.
  4. User received SMS; authentication failed;
    • Note point of failure in ticket: authentication failed: [method]


Authentication via DuoPush failed

  1. Repeat DuoPush authentication process
  2. User did not receive push notification;
    • Verify wireless connectivity (either WiFi or cellular).
    • Ensure the user is attempting DuoPush with primary (1st listed) device.
    • Turn the phone to airplane mode and then back to normal operating mode.
    • Turn off the WiFi connection and attempt to use the cellular data connection.
    • Verify device set to sync date and time automatically with the network.
    • Note point of failure in ticket.
  3. On occasion, the DuoMobile app DuoPush services get unsynchronized, so the user is advised to re-enroll their DuoMobile app.
    • Guide user to browse to the device management portal, which is the same site as the enrollment page,
    • ITS and Information Security recommend that all users authenticate via the mobile application using the passcode method, but the phone call method is equally effective.
    • Choose "Mange Devices"
    • Select "Enroll another device"
    • Open the DuoMobile App and select the "Key Plus" icon.
    • Scan the 2-D barcode in the Enrollment page with the mobile device.
    • It will re-enroll the DuoMobile app.
  4. User received push notification; authentication successful;
    • Note point of failure in ticket: user error.
  5. User received push notification; authentication failed;
    • Note point of failure in ticket: authentication failed: [method]

Activate + Setup + Install

Self-enrollment process

return to index

  1. Gather resolution information and enter it into the ticket.
  2. Walk user through process below.


User will need to have a computer connected to the internet and smartphone, cellphone, or landline phone to complete this process.

  1. Download the DuoMobile application and launch the app.
    • Add a mobile phone to your DuoSecurity profile.
      • With laptop or desktop computer, browse to the DuoSecurity self-enrollment page.
      • Type in CWRU network ID and password when prompted (SSO page).
      • Click the Start Setup button.
      • Choose Mobile Phone as device type. Click Continue.
      • Choose country from the dropdown box (“United States” is default).
      • Enter 10-digit telephone number.
      • Check the box marked “(###) ###-#### is the correct phone number” and click Continue.
      • Choose phone operating system (iOS or Android). Click Continue.
      • Select I have DuoMobile installed and click Continue.
    • Add the CWRU account to your mobile application
      • On your mobile phone, open the DuoMobile application and tap the + or Add account button. Then tap Scan Barcode.
      • Use your mobile phone to scan the barcode shown on your computer screen.
      • After an account titled “Case Western Reserve University” appears on your phone, click Continue on your computer.
    • Add a descriptive name.
      • Click the gray Actions menu to the right of the newly added number.
      • Choose Change Device Name… and add a descriptive title for the number (ex., My Work Mobile).
      • Click the green check mark to complete the name change.
    • Add a land line as a backup device
      • Click the gray Enroll another device button.
      • Choose Landline. Click Continue.
      • Enter 10-digit telephone number.
      • Click Done to save all changes and exit the device management interface.
    • Sign in with mobile phone to test and complete the enrollment.
      • Navigate to the Enrollment page.
      • Choose the Device and click the green Log In button.
      • A webpage titled “Self-Enrollment Success” will appear. You have successfully enrolled.

    Authentication process

    1. Gather resolution information and enter it into the ticket.
    2. Verify Cisco AnyConnect VPN client is installed. Obtain from vpnsetup.case.edu if needed.
    3. Verify that user has enrolled a mobile phone, it is on, connected to a wireless signal and charged.
    4. Recall that VPN only works from off-campus or when using the Case Guest wireless network.
    5. Choose directions that match the authentication method the user is attempting.
    6. ITS recommends the mobile phone and passcode method.
    7. Walk user through steps.
    8. Resolve ticket.


    Mobile phone and passcode (recommended)

    1. Launch the Cisco AnyConnect VPN client.
    2. Choose a supported VPN profile from the dropdown menu
    3. Enter your CWRU Network ID and password in the appropriate fields.
    4. Open the DuoMobile application on your mobile phone.
    5. Tap the key icon to generate a 6-digit passcode.
    6. Type the passcode in the “Second Password” field and click Connect to complete the authentication cycle.
    7. The VPN client will show the warning banner.
    8. Click Accept to start your VPN session.

    NOTE: Supported profiles display a second password field).


    Mobile phone and DuoPush


    1. Launch the Cisco AnyConnect VPN client.
    2. Choose a supported VPN profile from the dropdown menu
    3. Enter your CWRU Network ID and password in the appropriate fields.
    4. Type “push” into the “Second Password” field.
    5. The DuoMobile application will send an alert to your mobile phone.
    6. Tap View Now to display the alert.
    7. Tap Approve to complete the authentication cycle.
    8. The VPN client will show the warning banner.
    9. Click Accept to start your VPN session.

    NOTE: Tap the red Deny button and then choose “It seems fraudulent” when prompted if you did not initiate the login process. This will alert the Information Security team to a potential issue. An unsolicited push request indicates that your password may have been compromised.

    Authenticate with SMS passcodes

    1. Launch the Cisco AnyConnect VPN client.
    2. Choose a supported VPN profile from the dropdown menu
    3. Enter your CWRU Network ID and password in the appropriate fields.
    4. Type “sms” into the “Second Password” field and press the Enter key to send an authentication code to your default, SMS-capable device. Passcodes generated using this method expire in 5 minutes and may be used only once.
    5. Enter your your CWRU Network ID and password again.
    6. Type your passcode into the “Second Password” field and click Connect to complete the authentication cycle.
    7. The VPN client will show the warning banner.
    8. Click Accept to start your VPN session.

    NOTE: Using SMS with the VPN requires the user to login twice, once to get the SMS code, and a second time to login with the code sent by the system.


    Authenticate with telephone callback

    1. Launch the Cisco AnyConnect VPN client.
    2. Choose a supported VPN profile from the dropdown menu
    3. Enter your CWRU Network ID and password in the appropriate fields.
    4. Type “phone” in the “Second Password” field.
    5. Answer your phone and press any key to complete the authentication cycle.
    6. The VPN client will show the warning banner.
    7. Click Accept to start your VPN session.


    NOTE: If multiple phones are enrolled, type “phone” and then the number that represents its place in the order of enrolled devices.

    Example: Three devices are enrolled in this order; an office phone, an Android tablet and a mobile phone. To place an authentication call to the mobile phone, type “phone3” in the “Second Password” field and click Connect. To authenticate using the office phone, type “phone” in the “Second Password” field. The primary device is considered the first phone.

    Program overview

    return to index

    1. Gather resolution information and enter it into the ticket.
    2. Multi-factor authentication (also, two-factor authentication, two-step verification, 2FA) provides additional layers of security to protect user data in the case of password theft.
    3. DuoSecurity two-factor authentication enhances the security of your CWRU accounts by using your phone or connected mobile device to verify your identity. This prevents anyone but you from accessing your CWRU accounts, even if they know your password.
    4. CWRU provides two-factor authentication for VPN sessions, and plans to integrate two-factor authentication with other services in the future.
    5. Authentication via DuoSecurity is available to any CWRU user with valid credentials on an opt-in basis.
    6. For more information regarding program details including; roadmaps, external use cases, policy matters and participation requirements, users are advised to contact the ITS Information Security Office.
    7. Resolve ticket.
