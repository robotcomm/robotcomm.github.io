Service Description

The Web Content Management (WCM) system is software platform that provides certain core capabilities to end users,

  • Rapidly develop and update standards-compliant web pages
  • Leverage reusable resources (templates, graphic elements, etc.) stored in a digital asset library

The university has implemented this system to promote cohesive, modern web-presence. To aid user adoption, the project team has completed an initial set of page layouts and offers on-demand training classes.

Use this article when:

  1. An employee or student delegate has questions about updating a university website or subordinate content (e.g., departmental website or micro-site),
  2. An educator or student has questions about updating an affiliated organization website (e.g., a fraternity, Society for the Advancement of Technical Writers),
  3. Any user calls to request additional information about a timeline for his or her website,
  4. Any user requests additional information regarding the Web Content Management product or training,
  5. Any user requests a new website,
  6. Any user requests to delete an existing website,
  7. Any user requests to change owners of existing website.

NOTE: The Web Content Management system should not be referred to in a shortened form or by an acronym, and should NEVER be referred by vendor name (T4).

Tools for Resolution

Required Information

  • Name,
  • CWRU Network ID,
  • Contact information (preferred and alternate),
  • Problem or Request description
  • Department, Organizational Unit (OU) or Business Unit (BU)
  • Support Category (T4, Web Services Engineering, or Other)

Additional Resources

Refer to the Site Migration Progress List for a list of current and future websites available in the Web Content Management environment.

If unsure or Site Migration Progess List is unavailable, use the URL Information Tool to determine who supports a particular website.

If a site is NOT supported by the T4WCMS resolver group, refer to www.case.edu (Web Services Engineering) script or contact the listed site owner.

NOTE: T4WCMS-supported sites are listed in the URL Information Tool as ‘maintained by or in T4’.

How to Escalate

Description and purpose

An escalation may be requested by an end user when he or she determines that a critical issue has not been acknowledged properly, remains unresolved for an unacceptable period of time or is severe enough that it requires atypical handling.

Escalation should not be confused with assignment to a resolver group. An escalated issue is typically brought to the attention of management within a second-level support group(s).

Who to contact for escalation

Issue escalation is not offered at this time. Assign all unresolved tickets to the T4WCMS resolver group.

Escalation procedures

Issue escalation is not offered at this time. Assign all unresolved tickets to the T4WCMS resolver group.
