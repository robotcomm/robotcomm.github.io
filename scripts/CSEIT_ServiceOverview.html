Service Description

Information Technology support at the Case School of Engineering is provided by both CSE IT and CWRU Information Technology Services (ITS).

All issues not resolved by a CWRU ITS Analyst should be assigned to the Engineering Computing resolver group for further investigation unless task steps instruct otherwise.

Tools for Resolution

Required Information

  • Name,
  • CWRU Network ID,
  • Contact information (preferred and alternate),
  • Location (building name and room number),
  • Problem or Request description
  • CC number (print issues only)

Additional Resources

Please refer to existing general-purpose, connectivity, ERP, SSO and Identity Management scripts as appropriate for additional troubleshooting steps.

All unresolved issues that are specific to the Case School of Engineering should be assigned to the Engineering Computing resolver group, unless task steps instruct otherwise.

Resolver Groups

Engineering Computing, Engineering Web Support, Mediavision (TEC issues only), Miscellaneous

Engineering Computing is available Mon.-Fri., 8 a.m - 5 p.m., EST (and evenings as scheduled). Inquiries placed before 5 p.m. will receive a same-day response. Inquiries placed outside of operating hours will receive a response on the next business day.

How to Escalate

Description and purpose

An escalation may be requested by an end user when he or she determines that a critical issue has not been acknowledged properly, remains unresolved for an unacceptable period of time or is severe enough that it requires atypical handling.

Escalation should not be confused with assignment to a resolver group. An escalated issue is typically brought to the attention of management within a second-level support group(s).

Who to contact for escalation

All escalations should be marked as such and assigned to the Engineering Computing resolver group.

Escalation procedures

Escalation is available Mon.-Fri., 8 a.m - 5 p.m., EST. Escalations are not available during planned or unplanned university closures, or during recognized university holidays.

Classify the reason for escalation as one of the following; Confirmed Outage (total or partial), Suspected Outage (total or partial), Abend, Slow response time, User Requirements not met, Other functional or Other technical.

Troubleshoot + Repair

Cannot connect to internet

  1. Gather resolution information and enter it into the ticket.
  2. Is user connecting from thin client / desktop or mobile computing device?
  3. If mobile (e.g., notebook, laptop, tablet) troubleshoot wireless connectivity, as appropriate.

  4. If thin client or desktop;

    1. Verify machine is physically connected to media converter or switch.
    2. Verify media converter or switch is powered on and functioning (e.g., green lights on and flashing).
    3. Verify that patch cable is not damaged and is fully seated at both ends.

  5. If thin client;

    1. Instruct user to log-out and back in.
    2. Test internet access.
    3. Instruct user to press and hold power button until machine powers off.
    4. Press power button to restart machine.
    5. Log in.
    6. Test internet access.
    7. Request remote access to machine and troubleshoot connectivity as appropriate.

  6. If desktop computer (not thin client);

    1. Request remote access to machine and troubleshoot connectivity as appropriate.
  7. Note troubleshooting steps taken in ticket, and result.
  8. Resolve ticket or assign to Engineering Computing resolver group.

NOTE: If machine is connected to center jack on Faceplate marked “RJ45” user will need to order a media converter from CSE IT. Note this in ticket and assign to Engineering Computing.

Cannot access SSO-protected resource

  1. Gather resolution information and enter it into the ticket.
  2. Instruct user to clear browser cache, close and restart.
  3. Troubleshoot per issue-specific script, as appropriate.
  4. Resolve, assign or escalate as instructed.

Cannot access CWRU credential-protected resource

  1. Gather resolution information and enter it into the ticket.
  2. Instruct user to clear browser cache, close and restart.
  3. Troubleshoot per application, platform or issue specific-script, as appropriate.
  4. Resolve, assign or escalate as instructed.

Network resource

  2. If thin client;

    3. Instruct user to press and hold power button until machine powers off.
    4. Press power button to restart machine.
    5. Log in.
    6. Test internet access.

  3. If desktop computer (not thin client);

    1. Request remote access to machine and troubleshoot connectivity as appropriate.
  4. Note troubleshooting steps taken in ticket, and result.
  5. Resolve ticket or assign to Engineering Computing resolver group.

Engineering Web Support

  1. Gather resolution information and enter it into the ticket.
  2. Assign ticket to Engineering Web Support resolver group.

Client computing issues

  1. Gather resolution information and enter it into the ticket.

  2. If thin client;

    1. Instruct user to log-out and back in.
    2. Instruct user to press and hold power button until machine powers off.
    3. Press power button to restart machine.
    4. Note any hardware error messages.
    5. Log in.
    6. Is error or problem replicated?

  3. If desktop computer (not thin client);

    1. Request remote access to machine and troubleshoot issues as appropriate.
  4. Note troubleshooting steps taken in ticket, and result.
  5. Resolve ticket or assign to Engineering Computing resolver group.

Application missing

  1. Gather resolution information and enter it into the ticket.

  2. If thin client;

    1. Instruct user to log-out and back in.
    2. Review for missing application.
    3. Instruct user to press and hold power button until machine powers off.
    4. Press power button to restart machine.
    5. Log in.
    6. Review for missing application.

  3. If desktop computer (not thin client);

    1. Request remote access to machine and attempt to locate application file.
  4. Note troubleshooting steps taken in ticket, and result.
  5. Resolve ticket or assign to Engineering Computing resolver group.

TEC

  1. Gather resolution information and enter it into the ticket.
  2. Troubleshoot per existing TEC script.
  3. If unresolved, assign or escalate to MediaVision resolver group as instructed.

Assistance with printing

  1. Gather resolution information and enter it into the ticket.
  2. Determine whether printer is managed by Meritech or CSE IT.
  3. If Meritech printer, obtain CC number from physical label and add to ticket. Assign ticket to Meritech resolver group.

  4. If CSE IT managed printer;

    1. How is user attempting to print (e.g., directly connected, usb stick, via server)?
    2. Troubleshoot general printer issues, as appropriate.
  5. Resolve ticket or assign to Engineering Computing resolver group.

Activate + Setup + Install

Create engineering website

  1. Gather resolution information and enter it into the ticket.
  2. CSE IT provides faculty an initial website framework setup within Drupal at no charge (approximately 10 hours of work).
  3. Drupal 6 (CMS) training and basic CMS support are also provided at no cost.
  4. Self-service training and information is available at the CSE IT website and Lynda.com.
  5. Assign ticket to Engineering Web Support resolver group.

Hardware purchase and installation

  1. Gather resolution information and enter it into the ticket.
  2. User may purchase media converters (copper to fiber) and switches (fiber endpoint to CAT network cable) from CSE IT.
  3. Engineering Computing will reply to all purchase and installation requests received before 5pm by close of business.
  4. Installation time will vary based on stock availability at the time of request, but will not exceed 3 business days.
  5. Assign ticket to Engineering Computing.

Request installation assistance (software, any)

  1. Gather resolution information and enter it into the ticket.
  2. If software title is available from Software Center, attempt to guide user through self-installation.
  3. Resolve ticket or assign to Engineering Computing.

Request application access

  1. Gather resolution information and enter it into the ticket.
  2. Assign ticket to Engineering Computing.

Request custom application development

  1. Gather resolution information and enter it into the ticket.
  2. CSE IT offers custom application development at a base rate of $50.00 USD / hr.
  3. Rates may vary on the basis of complexity.
  4. Assign ticket to Engineering Web Support.
