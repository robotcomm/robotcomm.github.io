Internet

Case Western Reserve students, faculty and staff have free access to the Internet through both wired and wireless networks.

ITS supports faceplate repair and installation at new construction sites through the Network Construction resolver group. A list of buildings that are only serviced by Network Construction is maintained by the resolver group.

Before resolving any faceplate call, check the list to determine whether the ticket should be assigned to the Network Construction or Faceplate resolver group.

Tools for Resolution

Required Information

Network Engineering must have IP address and MAC/Physical address to provide assistance. Do not create a ticket for Network Engineering without this information.
  • Name
  • CWRU Network ID
  • Contact information (preferred and alternate)
  • IP Address
  • MAC/Physical Address
  • Faceplate Number – 10 digit number on front panel of faceplate. format 221-01-03-000
  • Equipment Description - multiport switch "large grey box", or media converter "small, single port grey or blue box"
  • Location (building name, room number)
  • Problem or Request description ... Make sure to include:
    1. How long the problem has existed
    2. Whether user has connected to the wired network on campus before
  • Ask if lights are on the switch or converter – Also, are they blinking?
    If lights are not on, ask to check that the power cable is plugged in to the switch and the outlet.

Additional Resources

Should I assign this to Network Construction?
  1. Gather resolution information and enter it into the ticket.
  2. Check user location (building and room number) against column "B" (Building Name) and column "C" (Room number) of cross-reference spreadsheet.
  3. The building may be known by a common or alternate name (alias). If the building isn't listed in column "B" check to make sure that the location name does not appear in column "E" (Building Alias).
  4. If the location appears in either B+C OR E+C, then assign the ticket to Network Construction.
  5. Otherwise, troubleshoot, assign or escalate as described elsewhere in this article.

How to Escalate

Escalation procedures

Internet is a critical service.

Escalations are available whenever requested, including during planned university closures and recognized university holidays.

During unplanned university closures escalations may NOT be available.

Faceplate Installation is not considered a critical service. Escalation is available M-F between 8am and 5pm, EST. Escalations are not available during planned or unplanned university closures, or during recognized university holidays.

Classify the reason for escalation as one of the following; Confirmed Outage (total or partial), Suspected Outage (total or partial), Abend, Slow response time, User Requirements not met, Other functional or Other technical.

Description and purpose

An escalation may be requested by an end user when he or she determines that a critical issue has not been acknowledged properly, remains unresolved for an unacceptable period of time or is severe enough that it requires atypical handling.

Escalation should not be confused with assignment to a resolver group. An escalated issue is typically brought to the attention of management within a second-level support group(s).

NOTE: If essential research or job function is impaired due to an outage, follow critical service protocol.

Troubleshoot + Repair

Problems connecting to the internet

back to top

  1. Gather resolution information and enter it into the ticket.
  2. Use the System Lookup tool to check the status of the IP address on the network.
    • If status is Reg Disabled, follow Reg Disable scripts
    • If status is Quarantined, follow Quarantine scripts
  3. Check to see if user can access either an external site (speedtest.net) or an internal site (case.edu).
  4. IF user can connect to external websites ONLY, THEN
    1. User to clear their cache and cookies in their web browser, reboot the computer and try again.
    2. If the user is still unable to connect to case.edu sites, instruct them to run their anti-virus software. When finished, have them reboot their machine and try again.
  5. Verify that the user is logged in to VPN (See, VPN script).
  6. Troubleshoot the wireless or wired connection (see, Faceplate scripts) as appropriate.
  7. Make sure that the network card is enabled and has the most current driver.

    Find network card settings

    Windows

    1. From the start menu choose Settings > Control Panel.
    2. Windows 95/98/ME – double-click System. Then, choose Device Manager.
    3. Windows 2000/XP – double-click System. Then, choose Hardware > Device Manager.

    Apple

    1. OS X - From the apple menu choose Apple > System Preferences > Network.
    2. OS 8/9 From the apple menu choose Control Panels > TCP/IP.

  8. Note troubleshooting steps taken in the ticket.
  9. Assign unresolved tickets to Network Engineering resolver group.
  10. Resolve ticket.

How-to + Training

How do I connect to the internet?

back to top

  1. Gather resolution information and enter it into the ticket.
  2. Determine connection method (wired, wireless, wireless with VPN) and advise user per appropriate script ("Problems using the wireless connection", "Problems connecting to the Case Network" and var. VPN scripts, respectively).
  3. After user is connected, walk user through opening a web browser. Verify that user can reach internal and external websites.
  4. Assign unresolved tickets to the DSS (staff/faculty) or ITS CARE Center (students) resolver group.
