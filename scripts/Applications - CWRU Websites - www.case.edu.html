Service Description

www.case.edu is the main home page for Case Western Reserve University (CWRU). Numerous websites are accessible through www.case.edu.

CWRU allows users to operate websites on the campus network using URLs beneath the case.edu domain.

Tools for Resolution

Required information

  • Name,
  • CWRU Network ID,
  • Contact information (preferred and alternate),
  • Problem or Request description,
  • URL,
  • Support category (Yes, No or T4),
  • Browser type,
  • Applicable error message(s)

Additional resources

How to Escalate

Description and purpose

An escalation may be requested by an end user when he or she determines that a critical issue has not been acknowledged properly, remains unresolved for an unacceptable period of time or is severe enough that it requires atypical handling.

Escalation should not be confused with assignment to a resolver group. An escalated issue is typically brought to the attention of management within a second-level support group(s).

Escalation procedures

www.case.edu is considered a critical service.

Escalations are available whenever requested, including during planned university closures and recognized university holidays.
During unplanned university closures escalations MAY not be available.

Classify the reason for escalation as one of the following; Confirmed Outage (total or partial), Suspected Outage (total or partial), Abend, Slow response time, User Requirements not met, Other functional or Other technical.

Use published schedule and contact information to route P0 and P1 incidents to the appropriate resolver group and current Ops Manager On Call .

I am having difficulty accessing www.case.edu or a site with a case.edu domain.

return to index

  1. Gather resolution information and enter it into the ticket.
  2. Verify that the user has the most recent version of their web browser.
  3. Verify that the user is typing in the correct address.
  4. Determine scope of user issue.
    • single case.edu site; Treat the incident as a potential outage if you are also unable to reach single (user-provided) internal website under the case.edu domain.
    • multiple case.edu sites; Treat the incident as a network outage if you are also unable to reach several CWRU websites.
    • internal and external sites; Troubleshoot user connectivity.
  5. Note all troubleshooting steps taken in ticket details.
  6. Resolve the ticket or forward to Web Services Engineering resolver group.

NOTE: CWRU websites are not tested for compatibility with IE6.

I am receiving an error message on a website with a case.edu domain

return to index

  1. Gather resolution information and enter it into the ticket.
  2. Verify that the user has the most recent version of their web browser.
  3. Verify that the user is typing in the correct address.
  4. Determine scope of user issue.
    • single case.edu site; Treat the incident as a potential outage if you receive the same error OR are also unable to reach single (user-provided) internal website under the case.edu domain.
    • multiple case.edu sites; Treat the incident as a network outage if you are also unable to reach several CWRU websites.
    • internal and external sites; Troubleshoot user connectivity.
  5. Use the URL Information Tool to determine who supports the website.
  6. Refer to resolution information if not supported by Web Services Engineering, unlisted, support contact is unlisted, or tool is inaccessible.
  7. Advise user that site is supported by Web Services Engineering OR provide user with site administrator contact.
  8. Note all troubleshooting steps taken in ticket details.
  9. Resolve the ticket or forward to Web Services Engineering resolver group.

How do I set up an account on case.edu?

return to index

  1. Gather resolution information and enter it into the ticket.
  2. Advise user of eligibility policy. Copy text in email if user requests.
  3. Direct user to account request form.
  4. Resolve the ticket.

Elegibility policy
Any official organization within the University is eligible for space on the main campus web server (www.case.edu). This includes, but is not necessarily restricted to, administrative departments and offices, academic departments, student organizations, Greek organizations, housing councils, and research groups.

In each case, an Project Inclusion Request (APIR) form must be filled out. This form must be completed by an individual with proper authority; for example, a department chair, the head of an administrative office, or a student group’s faculty advisor. The authorizer must be a full-time member of the Case staff or faculty.

Organizations which are affiliated with the University may be able to obtain space on www.case.edu. In order to do so, the affiliated organization will need to find a department of the University that is willing to sponsor the organization’s inclusion. If a University department withdraws its sponsorship for any reason, the affiliated organization’s material must be removed if another sponsor is not found. Note that all of the affiliated organization’s material housed on the www.case.edu server must comply with the Case Western Reserve University Acceptable Use of Computing and Information Technology Resources policy.

How do I update a webpage using WebDAV?

return to index

  1. Gather resolution information and enter it into the ticket.
  2. Access the WebDAV url by typing: https://www.case.edu:8000/[network_id]/[group_name]
  3. Log in with Case User ID and password.
  4. Direct the user to the web authoring tools found on help.case.edu.
  5. Resolve the ticket.

NOTE:
[network-id]: Case Network ID
[group_name]: account name of the site the user wants to update

If the user wishes to know if he or she is maintainer of an account, or what the group name is, ask the user to navigate to the My Account tool in the APAS toolkit.

How do I change the maintainer of my site?

return to index

  1. Gather resolution information and enter it into the ticket.
  2. Users may edit an existing Aurora account via the APIR form.
  3. Changes to organizational and departmental accounts must be reported to ITS AID.
  4. If listed account maintainer is no longer associated with it, steps will be taken to remedy (up to and including deactivation of access to the account).
  5. Resolve the ticket.
